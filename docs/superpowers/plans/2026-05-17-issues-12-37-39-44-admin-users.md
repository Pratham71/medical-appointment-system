**Goal:** Complete backend/frontend routing support for staff and patient-equivalent roles, admin user deactivation, and email notification hooks for appointment/document events.

**Architecture:** Keep the backend layered as routes -> services -> repositories -> queries -> db. Patient-equivalent roles reuse the existing `students` profile table. Email dispatch stays environment-driven and defaults to disabled in development so local work does not require SMTP credentials.

**Tech Stack:** FastAPI, Pydantic, raw MySQL SQL query modules, Next.js static routing checks, pytest, ruff.
